# 冷启动愿景规划设计规格

- 日期：2026-04-28
- 状态：已确认方向，待进入原型实施计划
- 对应原型：`prototype/index.html`
- 对应主文档：`产品需求设计文档.md`

## 1. 背景

现有冷启动只覆盖“输入季度目标 -> AI 拆成本周行动”的快速规划路径，适合低门槛体验，但缺少产品长期叙事中的“五年愿景”。这会让年度成长树只像一个任务反馈容器，而不是从长期自我出发的成长系统。

本轮确认采用“愿景推荐路径”：首次进入时，App 先弹窗询问用户希望怎样开始规划，主按钮推荐深度愿景规划，副按钮保留快速规划。

## 2. 设计目标

1. 补回“五年后想成为怎样的人”的愿景仪式感。
2. 不牺牲冷启动低门槛，保留一键进入现有快速规划的出口。
3. 让 AI 的拆解从“季度目标”升级为可从“长期愿景”逐层倒推。
4. 当前版本不把五年愿景做成主 App 常驻展示，避免产品变重。

## 3. 入口方案

采用 B 方案：愿景推荐路径。

首弹窗视觉结构：

```text
推荐路径
先看见五年后的自己

写下一句话就好。它不会变成压力，只会成为今年规划的北极星。

[体验深度愿景规划]
[先快速规划这个季度]
```

设计意图：

- 主按钮放“体验深度愿景规划”，强化产品的长期感。
- 副按钮放“先快速规划这个季度”，给不想深思的用户一个轻出口。
- 文案避免“请填写五年规划”这类表单感，强调“一句话就好”。

## 4. 两条路径

### 4.1 快速规划

快速规划沿用当前冷启动流程：

1. 询问“这个季度，你最想推进的一件事是什么？”
2. 用户输入季度目标。
3. AI 拆解本周重点和今日清单建议。
4. 用户进入今日清单或能量页，体验现有核心闭环。

这条路径服务于“我现在只想先用起来”的用户。

### 4.2 深度愿景规划

深度路径采用逐层倒推：

1. 五年愿景输入：询问“五年后，你希望自己成为一个什么样的人？”
2. 愿景认可：App 表达认可，并说明愿景不是压力，只是方向。
3. 年度 OKR 草案：AI 基于愿景生成今年的年度 OKR，用户可手动修改。
4. 季度 OKR 拆解：AI 将年度 OKR 拆成 4 个季度 OKR，用户可逐条编辑。
5. 首月 4 周承诺：AI 继续拆解第一个月，即 4 周承诺，用户确认或修改。
6. 天计划生成：AI 生成第一天的行动清单，进入现有清单 / 能量闭环。

## 5. 关键交互与文案

愿景输入页：

```text
五年后，你希望自己成为一个什么样的人？

可以很具体，也可以很模糊。
比如：身体强健，有稳定创造力，靠自己的作品获得自由。
```

认可反馈：

```text
我记住了。
这个愿景不用完美，它只是我们今年往前走的方向。
接下来，我会先帮你把它整理成今年的 OKR。
```

年度 OKR 页：

```text
这是我先为今年整理的一版方向。
你可以直接改掉不准确的地方。
```

季度 OKR 页：

```text
我们把今年分成四段走。
每一季只需要有足够清楚的重点，不需要把未来锁死。
```

首月 4 周承诺页：

```text
先只看接下来的四周。
更远的计划，等你真的走到那里再调整。
```

## 6. 数据与状态

新增概念：五年愿景。

- 作用：深度规划路径的上游语境，用于生成年度 OKR、季度 OKR、首月 4 周承诺。
- 当前版本展示范围：只在冷启动深度规划流程中展示。
- 当前版本不做：常驻首页展示、成长树详情展示、历史愿景版本管理。

新增规划层级：

```text
五年愿景 -> 年度 OKR -> 4 个季度 OKR -> 首月 4 周承诺 -> 天计划
```

快速规划层级保持：

```text
季度目标 -> 本周重点 -> 今日清单 -> 每日能量
```

## 7. 原型落点

`prototype/index.html` 需要补充：

1. 冷启动首弹窗：推荐“体验深度愿景规划”，副按钮“先快速规划这个季度”。
2. 深度规划 5 个关键页面：愿景输入、愿景认可、年度 OKR、季度 OKR、4 周承诺。
3. 编辑体验：年度 OKR、季度 OKR、4 周承诺均可手动修改或确认。
4. 结束状态：确认 4 周承诺后进入 AI 生成的天计划，也就是现有今日清单/能量闭环。

## 8. 不做范围

1. 不做五年愿景常驻展示。
2. 不做复杂年度计划管理后台。
3. 不做多版本愿景对比。
4. 不做 AI 聊天式愿景访谈。
5. 不把深度路径设为强制流程。

## 9. 验收标准

1. 首次进入能看到两个路径选择，视觉上推荐深度愿景规划。
2. 选择快速规划时，仍可完成原有冷启动流程。
3. 选择深度愿景规划时，用户能依次完成愿景、年度 OKR、季度 OKR、4 周承诺、天计划。
4. 每一层 AI 生成结果都允许用户修改或确认。
5. 文案保持“知性温柔的大姐姐”人格，不催促、不评判、不制造压力。
6. 主产品文档与原型流程保持一致。
