## 1. 文档定位

本目录承载个人成长管理 App 的技术架构设计，覆盖端侧、服务端、数据、AI、部署运维、技术风险和演进路线。

产品需求、用户流程、交互规则、产品人格和原型说明继续以 `产品需求设计文档.md` 为准。技术架构文档只回答“如何实现、如何分层、如何扩展、如何控制风险”。

## 3. MVP 技术原则

1. 移动端优先跑通体验闭环，不把首期复杂度投入到后台管理或多端扩张。
2. 后端采用模块化单体，保持部署简单，但保留清晰业务边界。
3. AI 只在规划和重新规划节点出现，不成为常驻聊天系统。
4. 端侧保留弱网可用能力，核心记录先能本地读写，再做云端同步。
5. 数据结构先服务季度目标、本周重点、今日清单、每日能量、周结算和成长树，不为远期假设过度设计。
6. 动画和成长树属于核心体验，不是装饰，因此前端架构要给自绘、动效和性能预留空间。
7. 鸿蒙在 MVP 阶段只承诺华为 Android 兼容设备可用，HarmonyOS NEXT 原生不进入首期硬约束。

## 4. 文档维护规则

当产品需求、原型或实际实现发生变化时，需要同步检查本目录：

1. 页面流程变化优先同步 `02-前端架构设计.md`。
2. AI 场景、结构化输出或拆解规则变化优先同步 `04-数据与AI架构.md`。
3. 新增后端业务模块或接口边界变化优先同步 `03-后端架构设计.md`。
4. 部署方式、环境变量、数据库迁移和监控策略变化优先同步 `05-部署与运维方案.md`。
5. 如果实际实现偏离本目录结论，需要在对应章节补充“当前实际方案”和“后续修正条件”。
