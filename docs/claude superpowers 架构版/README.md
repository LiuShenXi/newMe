- 文档状态：MVP 架构基线

## 1. 文档定位

本目录承载个人成长管理 App 的技术架构设计，覆盖端侧、服务端、数据、AI、部署运维、技术风险和演进路线。

产品需求、用户流程、交互规则、产品人格和原型说明以 `产品需求设计文档.md` 为准。手动 OKR 链路以 `docs/superpowers/specs/2026-04-29-manual-okr-design.md` 为准。技术架构文档只回答"如何实现、如何分层、如何扩展、如何控制风险"。

## 2. 阅读顺序

1. `00-技术架构总览.md` — 全局架构、系统边界、核心链路、模块地图。
2. `01-技术选型决策.md` — React Native、Expo、Zustand、NestJS、PostgreSQL、Prisma、AI 后端封装等选型。
3. `02-前端架构设计.md` — RN 工程分层、状态管理、本地数据、动画和页面模块。
4. `03-后端架构设计.md` — NestJS 模块边界、接口形态、事务、鉴权、错误处理和测试策略。
5. `04-数据与AI架构.md` — 核心数据模型、同步、AI 编排、结构化输出、成本和质量控制。
6. `05-部署与运维方案.md` — Docker 单体部署、环境配置、CI/CD、备份、日志、监控和发布。
7. `06-技术风险与演进路线.md` — 关键风险、缓解策略、阶段性演进条件和决策门槛。

## 3. MVP 技术原则

1. 移动端优先跑通体验闭环，不把首期复杂度投入到后台管理或多端扩张。
2. TypeScript 全栈，前后端类型共享，最大化 AI 编码效率。
3. 后端采用模块化单体，保持部署简单，但保留清晰业务边界。
4. AI 只在规划和重规划节点出现，不成为常驻聊天系统。模型无关，灵活切换。
5. 端侧保留弱网可用能力，核心记录先能本地读写，再做云端同步。
6. 冷启动支持三条路径（深度愿景、快速规划、手动 OKR），手动路径允许跳过和自下而上使用。
7. 动画和成长树属于核心体验，前端架构要给自绘、动效和性能预留空间。
8. UI 实现以 `prototype/index.html` 终稿静态原型为参照基准。

## 4. 文档维护规则

当产品需求、原型或实际实现发生变化时，需要同步检查本目录：

1. 页面流程变化优先同步 `02-前端架构设计.md`。
2. AI 场景、结构化输出或拆解规则变化优先同步 `04-数据与AI架构.md`。
3. 新增后端业务模块或接口边界变化优先同步 `03-后端架构设计.md`。
4. 部署方式、环境变量、数据库迁移和监控策略变化优先同步 `05-部署与运维方案.md`。
5. 如果实际实现偏离本目录结论，需要在对应章节补充"当前实际方案"和"后续修正条件"。
