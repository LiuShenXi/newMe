# NewMe 视觉设计语言文档

## 1. 设计主题

### 主题名称

**静谧森林 × 赛博能量**

### 一句话定义

NewMe 的视觉语言是一片安静、深邃、带有微弱能量流动的未来森林：它不是游戏界面，也不是传统效率工具，而是一个帮助用户每天为长期成长「注入能量」的个人空间。

### 核心视觉关键词

* 静谧
* 神秘
* 高级
* 克制
* 深色森林
* 微光能量
* 长期生长
* 温柔陪伴
* 低干扰正反馈
* 轻赛博，而非重科幻

---

## 2. 设计目标

NewMe 的界面设计要服务于三个产品目标：

1. **降低思考负担**

   * 用户进入 App 时不应该感到复杂、压迫、需要管理很多东西。
   * 页面信息必须少而准，每一屏只突出一个主动作。

2. **强化每日正反馈**

   * 每天记录推进度时，要有一个明确、轻微但令人愉悦的反馈瞬间。
   * 能量球、果实、树的成长不是装饰，而是产品闭环的一部分。

3. **建立长期成长感**

   * 用户需要感到：今天的记录不是孤立的，而是在为一棵年度成长树持续积累。
   * 视觉上要表达「慢慢生长」「被记住」「不会轻易消失」。

---

## 3. 品牌气质

### 人格气质

产品人格是：**知性温柔的大姐姐**。

视觉上不能过度可爱，不能像儿童游戏，也不能像冷冰冰的企业效率软件。它应该像一个安静、可靠、审美成熟的陪伴者。

### 情绪基调

| 维度  | 设计表达                 |
| --- | -------------------- |
| 温柔  | 柔和光晕、低对比卡片、圆润边角      |
| 知性  | 克制排版、清晰层级、少装饰        |
| 神秘  | 深色背景、雾感、森林暗部、局部微光    |
| 高级  | 大留白、低饱和色、细描边、少而精的动效  |
| 未来感 | 青绿色能量光、玻璃质感、微网格、流动光线 |

### 不要变成什么

* 不要变成重度游戏 UI。
* 不要变成赛博朋克霓虹灯牌。
* 不要变成普通 Todo App。
* 不要变成 AI 聊天工具。
* 不要变成复杂数据看板。
* 不要用大量卡通贴纸、奖章、金币、经验值。

---

## 4. 视觉母题

NewMe 的视觉系统由四个母题组成：

### 4.1 森林

代表长期、自然、静默、生长。

适用场景：

* 背景氛围
* 成长树页
* 冷启动页
* 年度回看
* 季度阶段变化

设计规则：

* 森林不应被画得过于写实复杂。
* 更适合使用抽象暗部、雾感、树影、枝干轮廓。
* 森林是空间氛围，不是装饰插画堆叠。

### 4.2 能量

代表每日推进、注入、正反馈。

适用场景：

* 能量页首页
* 今日推进度确认
* 能量球动效
* 每日记录成功反馈

设计规则：

* 能量色以青绿、电光青为主。
* 能量表现应该柔和、流动、内敛。
* 不要使用过强的红蓝霓虹、扫描线、故障风。

### 4.3 果实

代表每周结果、可回看的收获。

适用场景：

* 周结算
* 成长树页
* 时间胶囊

设计规则：

* 果实是「一周努力的凝结」，不是普通奖章。
* 亮度、饱满度、通透度可随周结果变化。
* 低分果实可以暗淡，但不应惩罚或羞辱用户。

### 4.4 荣誉层

代表季度达标后的永久积累。

适用场景：

* 季度结算
* 成长树荣誉效果
* 年度回看

设计规则：

* 荣誉层只增不减。
* 视觉应像树获得一层更深的生命力，而不是贴一个奖牌。
* 可用发光、流光、微粒、树冠辉光表达。

## 5. 色彩系统

### 5.1 主色调

NewMe 以极深绿色和黑色作为主背景，用低饱和青绿色作为功能光源，用金色作为果实和收获提示。

| 色彩角色                 |        色值 | 用途            |
| -------------------- | --------: | ------------- |
| Obsidian Black / 黑曜石 | `#030606` | App 外层背景、最深底色 |
| Deep Forest / 深林绿    | `#07110F` | 页面主背景         |
| Moss Dark / 暗苔绿      | `#0A1613` | 底部导航、深色容器     |
| Forest Mist / 森林雾绿   | `#12352B` | 次级背景、弱装饰      |
| Energy Cyan / 能量青    | `#55FFE0` | 核心能量、进度、选中态   |
| Soft Cyan / 柔青       | `#9DFFF0` | 高亮文字、描边、发光    |
| Life Green / 生命绿     | `#70FFBE` | 成长、完成、轻积极反馈   |
| Fruit Gold / 果实金     | `#E9D58A` | 周果实、收获、仪式感    |
| Amber Core / 暖金芯     | `#D8AE46` | 果实内部、结算重点     |
| Text White / 主文字     | `#F4FFF9` | 一级文本          |
| Text Mist / 雾白灰      | `#B9C9C3` | 二级文本          |
| Text Dim / 暗灰绿       | `#6F817A` | 说明文本、禁用态      |

### 5.2 背景渐变

推荐背景结构：

```css
background:
  radial-gradient(circle at 50% 12%, rgba(37,255,219,.14), transparent 30%),
  radial-gradient(circle at 20% 85%, rgba(120,255,175,.10), transparent 35%),
  linear-gradient(180deg, #091411 0%, #060B0A 55%, #030605 100%);
```

使用原则：

* 顶部可以有轻微青色光源。
* 底部可以保留深黑压暗。
* 页面整体亮度不要太高。
* 光源只服务于视觉焦点，不要平均铺满。

### 5.3 功能色规则

| 状态       | 推荐表达               |
| -------- | ------------------ |
| 选中       | 青绿色弱光 + 细描边        |
| 完成       | 生命绿或能量青，不用大面积纯绿    |
| 重点       | 果实金或柔青，避免红色警示感     |
| 警告       | 暖金或低饱和琥珀，不使用强红     |
| 禁用       | 深灰绿、低透明度           |
| 失败 / 未完成 | 不使用刺眼红色，采用暗淡、低亮度表达 |

## 6. 字体与排版

### 6.1 字体建议

中文：

* 系统默认：`PingFang SC` / `HarmonyOS Sans SC` / `Noto Sans SC`
* 若做品牌视觉稿，可考虑更高级的屏显黑体，但产品内仍以系统字体优先。

英文与数字：

* `Inter`
* `SF Pro Display`
* `DIN` 可用于百分比、能量数值、统计数字。

### 6.2 字重规则

| 场景    | 字重        |
| ----- | --------- |
| 页面标题  | 600       |
| 卡片标题  | 500 / 600 |
| 正文    | 400       |
| 辅助说明  | 400       |
| 数字能量值 | 300 / 400 |

不要大量使用粗体。高级感来自克制，而不是加粗。

### 6.3 字号系统

| 层级            |    字号 | 用途         |
| ------------- | ----: | ---------- |
| Display       | 48–56 | 能量百分比、关键数据 |
| Page Title    | 26–30 | 页面标题       |
| Section Title | 16–18 | 模块标题       |
| Body          | 14–15 | 正文内容       |
| Caption       | 11–12 | 标签、说明、状态   |
| Micro         | 10–11 | 次级辅助信息     |

### 6.4 字距规则

* 英文小标题可使用较大字距：`letter-spacing: 0.24em–0.46em`。
* 中文正文不额外拉字距。
* 大数字可使用负字距：`letter-spacing: -0.04em` 到 `-0.08em`。

## 7. 布局系统

### 7.1 页面结构

移动端页面遵循三段式：

1. 顶部：状态 / 页面标题 / 当前场景图标
2. 中部：本页唯一主舞台
3. 底部：辅助信息 / 主按钮 / 底部导航

### 7.2 间距规则

| 间距 Token  | 数值 | 用途          |
| --------- | -: | ----------- |
| `space-1` |  4 | 图标与文字细间距    |
| `space-2` |  8 | 标签内边距、小元素间距 |
| `space-3` | 12 | 列表项内部间距     |
| `space-4` | 16 | 卡片内边距       |
| `space-5` | 20 | 模块间距        |
| `space-6` | 24 | 页面左右边距      |
| `space-8` | 32 | 大区块间距       |

页面左右边距推荐：`24px`。

### 7.3 圆角规则

NewMe 的圆角应该柔和但不过度可爱。

| 场景           |    圆角 |
| ------------ | ----: |
| App 外框 / 大容器 | 40–46 |
| 主卡片          | 24–34 |
| 列表项          | 16–20 |
| 按钮           | 16–22 |
| 标签           |   999 |
| 能量球 / 果实     |   999 |

### 7.4 卡片密度

每屏最多建议：

* 1 个主视觉区域
* 1–2 个主要信息卡片
* 1 个主操作按钮

不要在首页堆叠大量任务、统计、图表、AI 入口。

## 8. 组件设计规则

## 8.1 玻璃卡片 Glass Card

### 用途

承载重点信息、清单、计划节点、结算内容。

### 视觉规则

```css
background: rgba(255,255,255,0.045);
border: 1px solid rgba(185,255,224,0.10);
backdrop-filter: blur(20px);
box-shadow:
  inset 0 1px 0 rgba(255,255,255,0.08),
  0 24px 56px rgba(0,0,0,0.24);
border-radius: 28px;
```

### 使用原则

* 卡片背景必须低透明，不要实色块压住氛围。
* 边框要细，不要做白色粗描边。
* 阴影向黑色靠，不要做浅色浮层阴影。

---

## 8.2 能量球 Energy Orb

### 定位

能量球是首页主舞台，是用户每天记录推进度时的情绪高点。

### 视觉结构

1. 外层柔光光晕
2. 半透明球体边界
3. 内部暗色液体或能量雾
4. 底部气泡上涌
5. 中央百分比数字
6. 确认后短暂增强发光

### 色彩规则

| 百分比     | 视觉表现      |
| ------- | --------- |
| 0–30%   | 暗青、低亮、气泡少 |
| 31–60%  | 森林绿 + 柔青光 |
| 61–85%  | 明显青绿色发光   |
| 86–100% | 青绿 + 暖金高光 |

### 动效规则

* 充电涌入持续：1–2 秒。
* 百分比数字可以缓动跳动。
* 气泡从底部向上漂浮。
* 发光增强后自然回落。
* 不要做强烈爆炸、闪屏、震动。

---

## 8.3 底部导航 Bottom Tab

### 结构

固定 4 个 Tab：

1. 能量
2. 清单
3. 计划
4. 成长树

### 视觉规则

* 背景：深苔绿半透明。
* 选中态：青色弱光 + 圆角胶囊底。
* 未选中：暗灰绿文字和图标。
* 图标线宽：1.5–1.8。

### 禁忌

* 不要使用彩色多图标。
* 不要在 Tab 上添加红点催促。
* 不要让 AI 成为一个常驻 Tab。

---

## 8.4 Todo 列表项

### 定位

清单页是执行主场，但不是复杂任务管理器。

### 视觉规则

* 每条任务用轻卡片或行项目承载。
* 勾选完成后降低对比，不使用强烈删除线。
* 未完成任务保持清晰，但不制造压力。

### 状态表达

| 状态     | 表达             |
| ------ | -------------- |
| 未完成    | 深色行背景 + 暗描边    |
| 已完成    | 青绿弱填充 + 文本降级   |
| 当前重点相关 | 可加一枚轻标签        |
| AI 建议  | 使用非常轻的标识，不喧宾夺主 |

---

## 8.5 周计划时间轴

### 定位

计划页用于中频回顾，不是项目管理甘特图。

### 视觉规则

* 纵向时间轴。
* 每周一个节点。
* 当前周发光。
* 已过去周显示结果。
* 未来周使用虚线、低亮、轻透明。

### 信息密度

每个周节点只展示：

* 周编号
* 当前状态
* 该周重点摘要
* 完成结果或待执行状态

不要在时间轴中直接展开每日任务。

---

## 8.6 成长树

### 定位

成长树是年度回看空间，不是首页高频任务入口。

### 视觉规则

* 树要有生命感，但不需要复杂写实。
* 树冠可用暗绿色柔光块面。
* 树干可略带厚涂质感。
* 果实要有发光与透明感。
* 背景可有森林雾气、暗部、微粒。

### 阶段变化

| 阶段      | 视觉方向               |
| ------- | ------------------ |
| Q1 起始阶段 | 小树苗、光弱、空间安静        |
| Q2 成长阶段 | 树冠展开、枝干明显、少量果实     |
| Q3 丰茂阶段 | 叶量增加、果实更多、光感增强     |
| Q4 年度完成 | 树形完整、荣誉层明显、整体更有仪式感 |

---

## 8.7 果实

### 定位

果实是每周结果的视觉沉淀，也是时间胶囊入口。

### 视觉规则

* 用暖金或青金色，不要用普通红苹果。
* 果实亮度与周结果连续对应。
* 果实点击后弹出该周回顾卡片。

### 结果映射

| 周结果     | 果实状态         |
| ------- | ------------ |
| 0–30%   | 小、暗、低透明      |
| 31–60%  | 有形、有微光       |
| 61–85%  | 饱满、明显发光      |
| 86–100% | 通透、亮度高、有柔光外溢 |

---

## 9. 图标系统

### 风格

* 线性图标
* 圆角端点
* 线宽 1.5–1.8
* 不使用复杂填充图标
* 不使用过度游戏化图标

### 图标语义

| 功能    | 图标方向        |
| ----- | ----------- |
| 能量    | 闪电、能量流、圆形核心 |
| 清单    | 列表、勾选、路径    |
| 计划    | 时间轴、旗帜、路线   |
| 成长树   | 树叶、树、果实     |
| AI 拆解 | 微光、星点、柔和魔法感 |

### AI 图标规则

AI 不应该像聊天机器人头像，也不应该像客服入口。更适合表达为：

* 微光
* 星尘
* 轻微流线
* 「幕后生成」的感觉

---

## 10. 动效系统

### 动效原则

NewMe 的动效应该是：

* 慢
* 柔
* 有呼吸感
* 有延迟感
* 像能量流动，不像按钮炫技

### 推荐动效

| 场景     | 动效                 |
| ------ | ------------------ |
| 能量确认   | 气泡上涌 + 数字滚动 + 球体微亮 |
| 周结算    | 果实从树枝上慢慢成形并亮起      |
| 成长树    | 树冠微光呼吸、果实轻微闪烁      |
| 计划生成   | 微粒聚合成每周节点          |
| 页面切换   | 轻微淡入 + 上移 8px      |
| Tab 切换 | 胶囊底缓动滑入            |

### 动效参数建议

| 动效     |          时长 | 缓动                       |
| ------ | ----------: | ------------------------ |
| 普通点击反馈 |   120–180ms | ease-out                 |
| 页面进入   |   240–320ms | cubic-bezier(.2,.8,.2,1) |
| 能量充电   | 1000–2000ms | ease-in-out              |
| 果实生成   | 1600–2400ms | ease-in-out              |
| 呼吸光    | 3000–5000ms | ease-in-out infinite     |

### 禁忌

* 不使用高频闪烁。
* 不使用强烈弹跳。
* 不使用老虎机式奖励动效。
* 不使用大量粒子爆炸。
* 不让动效阻塞用户完成核心操作。

---

## 11. 插画与图形风格

### 推荐风格

* 暗色厚涂感
* 抽象森林空间
* 柔和边缘
* 低饱和色块
* 局部高亮
* 轻微赛博材质

### 树的风格

树可以更接近「厚涂奇幻概念图」，但界面中的树要被 UI 化处理：

* 形体简洁
* 层次清楚
* 不要细节过多
* 适合被果实点击
* 适合叠加季度荣誉光效

### 背景风格

背景不是一张完整风景图，而是一个氛围层：

* 暗部森林
* 雾气
* 若隐若现的树影
* 轻微网格或能量纹理
* 局部青绿色光源

## 12. 页面级设计规则

## 12.1 冷启动页

### 目标

先让用户感受到产品是什么，再引导他输入一个季度目标。

### 设计规则

* 首屏展示小树苗。
* 文案温柔，不要表单感。
* 输入框像一个安静的许愿空间，而不是任务创建框。
* AI 拆解按钮要轻，不要像强营销 CTA。

### 推荐文案

> 这是你今年的成长树，它会陪你走过四个季节。

> 这个季度，你最想推进的一件事是什么？

---

## 12.2 能量页

### 目标

展示本周推进状态，并让用户记录今日推进度。

### 设计规则

* 能量球是绝对主视觉。
* 本周重点推进概览作为打分参照物。
* 今日推进滑条放在能量球之后。
* 不出现长任务列表。
* 不出现 AI 聊天框。
* 不出现复杂统计。

### 信息优先级

1. 本周能量球
2. 本周重点推进概览
3. 今日推进度滑条
4. 确认今日能量按钮

---

## 12.3 清单页

### 目标

让用户知道今天做什么。

### 设计规则

* 默认只展示今日清单。
* 本周入口放右上角。
* 本周重点用轻标签或摘要呈现。
* 不按本周重点强行分组任务。
* 支持 CRUD，但视觉上保持轻。

### 信息优先级

1. 今日日期与简短说明
2. 今日 Todo
3. 添加任务
4. 本周 7 天概览入口

---

## 12.4 计划页

### 目标

展示季度 / 年度计划的宏观节奏。

### 设计规则

* 纵向时间轴。
* 当前周高亮。
* 每次只呈现 4 周计划重点。
* AI 重规划是辅助按钮，不抢主视觉。
* 点击周节点可以进入该周清单概览。

---

## 12.5 成长树页

### 目标

承载长期反馈与年度回看。

### 设计规则

* 树是主视觉。
* 果实是可点击时间胶囊。
* 统计信息只保留最关键的 2–3 项。
* 季度进度可以用很轻的进度条表达。
* 不把成长树页做成成就墙。

---

## 12.6 周结算页

### 目标

让用户温和回顾这一周，并确认最终周结果。

### 设计规则

* 有仪式感，但不要太游戏化。
* 先用一句温柔开场。
* 再展示每日能量曲线。
* 再展示建议周结果。
* 最后生成果实。

### 推荐节奏

1. 开场：这一周辛苦了。
2. 回顾：本周每天的推进情况。
3. 确认：建议周结果。
4. 生成：果实慢慢亮起。
5. 收束：安静地展示树与新果实。

---

## 13. 文案视觉规则

### 语气

文案要像温柔的陪伴者，而不是效率教练。

### 推荐表达

| 场景     | 推荐文案                    |
| ------ | ----------------------- |
| 每日提醒   | 今天过得怎么样？有空的话记录一下吧。      |
| 打分前提醒  | 看起来今天的清单还没看过，要不要先看看再打分？ |
| 周结算    | 这一周辛苦了，我们一起看看这周的收获吧。    |
| 长时间未打开 | 好几天没见了，最近还顺利吗？          |
| 季度达标   | 这个季度你真的很棒，树记住了你的努力。     |

### 避免表达

| 不推荐       | 原因      |
| --------- | ------- |
| 你已经落后了    | 制造压力    |
| 任务未完成     | 过度任务化   |
| 快来打卡      | 低质打卡产品感 |
| 恭喜升级      | 游戏化过重   |
| AI 助手为你服务 | 偏聊天工具感  |

## 17. 视觉验收标准

每一张页面设计稿都应该通过以下检查：

### 产品层检查

* 这一屏的唯一主动作是否清楚？
* 是否避免了复杂效率工具感？
* 是否符合「AI 在幕后」的原则？
* 是否保留了本周重点作为行动锚点？
* 是否让长期成长反馈可见？

### 视觉层检查

* 是否足够安静？
* 是否足够高级？
* 是否有森林感但不俗套？
* 是否有赛博能量感但不过度霓虹？
* 是否没有游戏 HUD 感？
* 光效是否集中在关键元素上？

### 交互层检查

* 今日记录是否足够轻？
* 周结算是否有仪式感但不打扰？
* 成长树是否能承载回看？
* 清单页是否仍然像执行工具，而不是目标管理大后台？

## 19. 总结

NewMe 的视觉设计不是为了让用户觉得「酷」，而是为了让用户愿意每天安静地回来一次。

它的核心不是刺激，而是积累；不是打卡，而是注入；不是任务数量，而是推进感；不是短期奖励，而是一棵会记住用户努力的年度成长树。

最终视觉判断标准：

> 当用户打开 App 时，他应该感觉自己进入了一个安静、神秘、可靠的私人森林空间。这里没有催促，没有审判，只有今天可以注入的一点能量，以及长期会被记住的成长痕迹。

## 20. 终稿静态 HTML 原型补充

最后更新：2026-04-26

### 20.1 原型产物

- 终稿 React 源稿：`终稿原型.jsx`
- 可直接打开版本：`prototype/index.html`
- 运行方式：无需 React、Tailwind、构建工具或本地服务，直接用浏览器打开 HTML 文件。

本轮已把 JSX 终稿迁移为单文件原生 HTML/CSS/JavaScript。静态版保留终稿的信息结构、页面状态和主要交互：冷启动、能量页、清单页、计划页、成长树、周结算、弹窗、重置与跳过冷启动。

### 20.2 设计基调

NewMe 的原型基调是深色、安静、带轻微生命感的个人成长工具。页面不做任务管理后台感，而是把“每日能量”和“年度成长树”作为主舞台，减少顶部标题和密集信息，让用户先感到推进，再查看细节。

核心感受：

1. 温柔但不催促
2. 沉浸但不游戏化过重
3. 低脑力规划，强即时反馈
4. 数据藏在需要时出现的入口里

### 20.3 视觉系统

#### 20.3.1 画布

- 外层背景：近黑绿色 `#030606`
- 手机底色：`#07110f`
- 手机尺寸：393 × 812，对齐 iPhone 14 Pro 原型尺度
- 背景纹理：深色径向光 + 22px 网格弱纹理

#### 20.3.2 颜色

- 主反馈色：青色 / 水绿色，用于能量球、进度条、当前状态
- 辅助正反馈：琥珀色，用于果实、周结算、成长收获
- 文字层级：
  - 主要文字：白色或 `#f1f5f9`
  - 次级说明：`#94a3b8`
  - 弱标签：低透明度浅绿或浅青

#### 20.3.3 组件语言

- 玻璃卡片：半透明白叠加、细描边、轻内高光
- 底部导航：四个固定 Tab，当前项使用浅青背景和内发光
- 能量球：中心大百分比、光晕、气泡、充能状态标签
- 成长树：主画面只展示树、果实和三个小入口，不把统计信息直接压在树上
- 弹窗：手机内部居中浮层，半透明黑遮罩 + 模糊

### 20.4 页面结构

#### 20.4.1 冷启动

1. 成长树开场
2. 输入季度最重要的一件事
3. AI 拆成本周行动
4. 进入 NewMe

冷启动强调“先用一件事跑通闭环”，不引导用户一次性配置复杂目标系统。

#### 20.4.2 能量页

一屏只保留四个核心元素：

1. 本周能量球
2. 本周进度概览
3. 今日能量条
4. 确认今日能量按钮

如果用户尚未看过清单就确认能量，弹出温和提醒，允许先看清单或继续注入。

#### 20.4.3 清单页

清单页默认展示今日事项，顶部用本周重点短标签提供方向感。支持：

- 勾选 / 取消勾选
- 添加今日小事
- 查看本周 7 天概览
- 从计划页选择某一周后带入“来自 Wxx 的计划”提示

#### 20.4.4 计划页

计划页用顶部切换区分两个视角：

- 月计划 · 按周
- 年计划 · 按 Q

月计划只展示最近 4 周，支持 AI 重规划。年计划只展示四个季度阶段，不下钻复杂年度路线图。

#### 20.4.5 成长树

成长树保持沉浸式主画面。树旁三个按钮进入详情：

- 果实
- 阶段
- 荣誉

点击果实打开时间胶囊弹窗，展示当周日期、结果百分比和回顾说明。

#### 20.4.6 周结算

周结算从右侧说明面板进入，手机内仍复用成长树导航态。流程保留：

1. 本周收获开场
2. 每日能量柱状图
3. 建议周结果
4. 确认并生成果实

### 20.5 静态 HTML 实现说明

`prototype/index.html` 使用原生状态对象渲染页面，不依赖 React。主要状态包括：

- `booted`：是否完成冷启动
- `tab`：当前页面
- `todos`：今日清单
- `weekRecords`：本周能量与清单完成记录
- `planView`：计划页月 / 年切换
- `treeDetail`：成长树详情页类型
- `showReminder` / `showWeek` / `selectedFruit`：弹层状态

交互通过事件委托绑定在 `document` 上，方便保持单文件结构。所有 CSS、HTML 模板和 JavaScript 都内联在同一个文件中，确保双击 HTML 即可体验。
