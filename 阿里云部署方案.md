## 1. 整体架构与这份方案覆盖的范围

你的项目长这样（已经用 Docker Compose 打包好了）：

```text
 用户手机 / 浏览器
       │ HTTPS
       ▼
┌──────────────────────────────┐
│  阿里云 ECS 一台（2 核 4G）   │
│                              │
│   Nginx (容器)  80/443 端口   │
│      │                       │
│      ▼                       │
│   NestJS API (容器) 3000      │
│      │                       │
│      ▼                       │
│   PostgreSQL 16 (容器) 5432   │
└──────────────────────────────┘
       │（出站）
       ▼
  智谱 GLM API（备用大模型）
```

这份方案会用**最省钱、最简单**的组合把服务跑起来：

- **服务器**：阿里云 ECS（云服务器）一台。
- **数据库**：跑在同一台服务器里的 Postgres 容器（先不买独立 RDS，省钱）。
- **反向代理**：`docker-compose.yml` 里自带的 Nginx 容器 + 宿主机上再套一层 Nginx 做 HTTPS（更稳）。
- **域名**：阿里云买一个 `.com`，并完成 ICP 备案（国内服务器强制要求，否则 80/443 端口会被拦）。
- **证书**：阿里云免费 SSL 证书（一年一签）。
- **文件存储**（可选）：头像、上传图片走 OSS，后面再接。
- **CDN**（可选）：用户多起来再开。

> 🚫 不涉及：Kubernetes、多可用区、灰度发布。这些对现在的阶段是过度设计。等 DAU 过万再说。

## 3. 买什么样的阿里云服务器（ECS）

### 3.1 直接给结论

| 项目 | 推荐配置 | 原因 |
| --- | --- | --- |
| 实例规格族 | **ecs.e-c1m2.large**（突发性能型 e，2 核 4G） | NestJS + Postgres + Nginx 三个容器，4G 内存是舒适下限。2G 会 OOM。 |
| 地域 | **华东 1（杭州）** 或 **华北 2（北京）** | 国内用户访问最快。选一个离你目标用户近的就行。 |
| 镜像 | **Ubuntu 22.04 64 位** | 社区资料最多，Docker 官方支持最好。 |
| 系统盘 | **ESSD Entry 40 GB** | Postgres 数据、Docker 镜像、日志都在这里。40G 够撑半年到一年。 |
| 带宽 | **按固定带宽 3 Mbps** | 上线初期足够；按流量计费在突发情况下会被打爆账单。 |
| 公网 IP | **分配** | 没公网 IP 就连不上。 |
| 购买时长 | **包年包月 1 年** | 比按量付费便宜 40%+。新用户首年有优惠。 |

### 3.2 一步一步怎么买

1. 浏览器打开 [aliyun.com](https://www.aliyun.com)，右上角「登录」，没账号就注册 + 实名认证（走支付宝人脸识别最快）。
2. 顶部导航「产品」→「云服务器 ECS」→ 点「立即购买」。
3. 计费方式选 **「包年包月」**。
4. 地域：挑 **华东 1（杭州）** 就行。
5. 实例：搜索框输入 `e-c1m2.large`，选 **2 vCPU 4 GiB** 那一款。看到右侧月价格大概在 80–110 元（新用户优惠下更便宜）。
6. 镜像：公共镜像 → Ubuntu → **22.04 64 位**。
7. 存储：系统盘 **ESSD Entry 40 GB**。数据盘暂时不买。
8. 下一步「网络和安全组」：
   - 分配公网 IPv4 地址 ✅
   - 带宽计费：**按固定带宽**，拉到 **3 Mbps**。
   - 安全组：选「新建安全组」，先放开 `22`（SSH）、`80`（HTTP）、`443`（HTTPS）。**一定不要**放开 `3000`、`5432`、`37200`，那些是内部端口。
9. 下一步「系统配置」：
   - 登录凭证：选 **密钥对**。点「新建密钥对」，取名如 `newme-prod`，**立刻把下载的 `.pem` 文件存到本地 `~/.ssh/newme-prod.pem`**（一旦关掉页面就再也下载不到了）。
   - 实例名称：`newme-prod-1`。
10. 「分组设置」默认即可。
11. 下一步「确认订单」：勾同意协议，支付。

> 💡 如果你是**第一次注册**阿里云，在「新用户特惠」页面经常能看到「ECS 99 元/年」的活动（通常是 2 核 2G）。如果你**能接受前三个月只放几个测试用户**，可以先用 99 元那台顶一下，等上线再切到 4G。但 **2G 跑 Postgres + NestJS 很紧**，建议直接上 4G。

### 3.3 记下这几样东西

买完之后在「ECS 控制台」能看到这台机器，复制保存：

- **公网 IP**：如 `47.xxx.xxx.xxx`
- **实例 ID**：`i-bp1xxxxxxxxxx`
- **SSH 密钥文件**：`~/.ssh/newme-prod.pem`（Mac 上执行 `chmod 600 ~/.ssh/newme-prod.pem`）

## 4. 买域名 + 备案（国内部署绕不开）

**为什么必须备案**：在中国大陆的服务器上对外开放 80 / 443 端口，必须完成 ICP 备案。不备案的域名指向国内 IP，会被运营商拦掉。

### 4.1 买域名

1. 阿里云顶部「产品」→「域名」→ 搜一个你想要的，比如 `newme-app.com`。
2. `.com` 一年大约 55–75 元。选 1 年就行。
3. 买完要**实名认证**（把域名绑到你阿里云账号的实名信息上）。一般 1 小时内通过。

### 4.2 ICP 备案

路径：阿里云控制台 → 顶部搜「备案」→「ICP 备案」→ 按向导填写。

- 主体：个人 or 企业（看你怎么注册的阿里云账号）。
- 需要材料：身份证正反面、手持身份证照、手机号、邮箱、居住地址、域名证书（点几下就能下载）。
- 阿里云会寄一块幕布给你，拍照上传（个人备案）。
- 通信管理局审核：**一般 3–20 个工作日**。这段时间不能在国内服务器上解析这个域名，但可以先把服务器准备好。

> ⏱ 备案是整个流程里最耗时的一环。**建议第一步先买服务器、第二步就同时发起备案**，两件事并行做。

### 4.3 备案过了之后做解析

阿里云控制台 → 云解析 DNS → 找到你的域名 → 添加解析：

| 记录类型 | 主机记录 | 解析线路 | 记录值 | TTL |
| --- | --- | --- | --- | --- |
| A | `@` | 默认 | 你的 ECS 公网 IP | 10 分钟 |
| A | `www` | 默认 | 你的 ECS 公网 IP | 10 分钟 |
| A | `api` | 默认 | 你的 ECS 公网 IP | 10 分钟 |

这样 `newme-app.com`、`www.newme-app.com`、`api.newme-app.com` 都会指向你的服务器。

## 5. 服务器首次登录与安全加固

打开 Mac 终端：

```bash
chmod 600 ~/.ssh/newme-prod.pem
ssh -i ~/.ssh/newme-prod.pem root@<你的公网IP>
```

第一次会问 `yes/no`，输 `yes`。进去之后就是你的服务器。

### 5.1 换成非 root 账户（强烈推荐）

```bash
# 在服务器上执行
adduser newme                 # 按提示设密码
usermod -aG sudo newme        # 加入 sudo 组
mkdir -p /home/newme/.ssh
cp ~/.ssh/authorized_keys /home/newme/.ssh/
chown -R newme:newme /home/newme/.ssh
chmod 700 /home/newme/.ssh
chmod 600 /home/newme/.ssh/authorized_keys
```

接着禁用 root 登录（可选但推荐）：

```bash
sudo sed -i 's/^PermitRootLogin .*/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart ssh
```

以后登录用：

```bash
ssh -i ~/.ssh/newme-prod.pem newme@<你的公网IP>
```

### 5.2 系统更新 + 防火墙

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y ufw
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable        # 输 y 回车
sudo ufw status
```

> 注意 ufw 规则和阿里云安全组是**两层**，两层都要放行才通。两层都只留 22/80/443。

### 5.3 调时区 + 开 swap

```bash
sudo timedatectl set-timezone Asia/Shanghai

# 开 2G swap，防止内存偶发性打满时被 OOM Killer 干掉
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h   # 确认 Swap 行不为 0
```

## 6. 安装 Docker 与 Docker Compose

在服务器上执行（Ubuntu 22.04 官方推荐脚本）：

```bash
# 1. 卸掉可能的旧版本
sudo apt remove -y docker docker-engine docker.io containerd runc

# 2. 安装 Docker 官方源
sudo apt update
sudo apt install -y ca-certificates curl gnupg lsb-release
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg \
  | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 3. 安装 Docker Engine + Compose 插件
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 4. 把 newme 账户加进 docker 组，免去每次 sudo
sudo usermod -aG docker newme
# 退出重登一次让 group 生效
exit
```

重新登进来验证：

```bash
ssh -i ~/.ssh/newme-prod.pem newme@<你的公网IP>
docker version
docker compose version
```

两个命令都能打印版本就算成功。

### 6.1 配镜像加速器

阿里云给每个账号发了一个**专属**的镜像加速器地址。进阿里云控制台搜索「容器镜像服务 ACR」→ 左边「镜像工具 → 镜像加速器」，复制你看到的 `https://xxxxxxxx.mirror.aliyuncs.com`，然后：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": ["https://xxxxxxxx.mirror.aliyuncs.com"],
  "log-driver": "json-file",
  "log-opts": { "max-size": "10m", "max-file": "3" }
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

`log-opts` 这段是限制容器日志大小，防止一个 bug 刷爆硬盘。

```bash
sudo apt install -y git
```

### 7.2 把代码克隆上来

你有两个选择：

**方案 A：用 GitHub/Gitee 仓库**（推荐）。把项目推到一个私有仓库，在服务器上用 deploy key 或 PAT 拉下来。

**方案 B：直接从本地 Mac 打包上传**。适合还没托管的情况：

```bash
# 在本地 Mac 上执行
cd /Users/shenxi/Desktop/WORK-SPACE/newMe
tar --exclude='node_modules' --exclude='.git' --exclude='test-results' \
    --exclude='apps/mobile/node_modules' \
    -czf /tmp/newme.tar.gz .
scp -i ~/.ssh/newme-prod.pem /tmp/newme.tar.gz newme@<你的公网IP>:~/
```

服务器上：

```bash
mkdir -p ~/newme && cd ~/newme
tar -xzf ~/newme.tar.gz
rm ~/newme.tar.gz
```

### 7.3 准备生产环境变量

服务器上在项目根目录新建 `.env.production`（Compose 会自动读同目录的 `.env`，所以我们把它叫 `.env`）：

```bash
cd ~/newme
cat > .env <<'EOF'
# ---------- 对外端口 ----------
# Nginx 容器映射到宿主机的端口。改成 80 这样直接用 http:// 访问
NGINX_PORT=80

# ---------- JWT ----------
# 用命令生成：openssl rand -hex 48
JWT_SECRET=换成你生成的48字节随机字符串
JWT_ACCESS_EXPIRY=15m
JWT_REFRESH_EXPIRY=30d

# ---------- AI 降级到智谱 ----------
AI_LOCAL_BASE_URL=http://127.0.0.1:4100/v1
AI_LOCAL_API_KEY=local-dev-key
AI_LOCAL_MODEL=local-qwen-coder
AI_FALLBACK_BASE_URL=https://open.bigmodel.cn/api/paas/v4
AI_FALLBACK_API_KEY=换成你自己的智谱key
AI_FALLBACK_MODEL=glm-4-flash
AI_PROVIDER_TIMEOUT_MS=60000
EOF

chmod 600 .env
```

生成 JWT_SECRET：

```bash
openssl rand -hex 48
# 把输出贴到 .env 里 JWT_SECRET 后面
```

> ⚠️ **`.env` 文件包含所有密钥，绝对不能提交到 git**。项目里 `.gitignore` 应该已经排除了。

### 7.4 改 Postgres 密码

默认的 `docker-compose.yml` 里密码是 `newme`，上线必须换掉。打开 `docker-compose.yml`，把 Postgres 相关部分改成：

```yaml
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: newme_prod
      POSTGRES_USER: newme
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    # ...其他不动
```

并在 API 的 environment 里把 DATABASE_URL 改成用变量：

```yaml
      DATABASE_URL: postgresql://newme:${POSTGRES_PASSWORD}@postgres:5432/newme_prod
```

然后在 `.env` 追加：

```bash
echo "POSTGRES_PASSWORD=$(openssl rand -hex 16)" >> .env
```

### 7.5 第一次启动

```bash
cd ~/newme
docker compose up -d --build
```

第一次会下载基础镜像 + 编译代码，**5–10 分钟**属正常。期间可以：

```bash
docker compose logs -f api      # 看 API 日志
docker compose ps               # 看容器状态
```

全部 `Up (healthy)` 就成功了。健康检查：

```bash
curl -fsS http://127.0.0.1/api/v1/health
# 期望看到 JSON，类似 {"status":"ok",...}
```

如果返回 200，说明后端已经通了。现在用浏览器访问 `http://<公网IP>/api/v1/health` 也能看到同样结果。

## 8. 配置 HTTPS（让用户用 `https://` 访问）

前置条件：**域名已经备案通过** + **DNS 已解析到你的 ECS IP**。用 `ping api.newme-app.com` 能看到正确的 IP 才能继续。

策略：在**宿主机**上跑一份 Nginx，做 HTTPS 卸载，然后反向代理到 Compose 里的 Nginx 容器。这样证书续签、后端重启互不影响。

### 8.1 先让 Compose 里的 Nginx 改监听到内网

编辑 `docker-compose.yml`，把 `nginx` 服务的端口映射从 `"${NGINX_PORT:-37200}:80"` 改成只暴露到本机：

```yaml
  nginx:
    image: nginx:1.27-alpine
    depends_on:
      - api
    ports:
      - "127.0.0.1:37200:80"   # 只本机能访问
    # ...
```

然后 `.env` 可以去掉 `NGINX_PORT` 那一行。重新启动：

```bash
docker compose up -d
```

现在 `curl http://127.0.0.1:37200/api/v1/health` 能通，但从外网访问 80 端口什么都没有——这正是我们想要的。

### 8.2 装宿主机 Nginx + Certbot

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

### 8.3 写一个站点配置

```bash
sudo tee /etc/nginx/sites-available/newme <<'EOF'
server {
    listen 80;
    server_name api.newme-app.com;      # 改成你的域名

    # Let's Encrypt 验证会走这条 location
    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    # 其他请求暂时先 301 到 https（证书签好之后再生效）
    location / {
        return 301 https://$host$request_uri;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/newme /etc/nginx/sites-enabled/newme
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx
```

### 8.4 申请证书

```bash
sudo certbot --nginx -d api.newme-app.com --email 你的邮箱@xxx.com --agree-tos --no-eff-email
```

Certbot 会自动改写上面那份 nginx 配置，加上 443、证书路径、HTTP→HTTPS 重定向。签完之后它还会每 90 天自动续签（安装时已经把 systemd timer 加好了）。

### 8.5 把 HTTPS 请求转发给 Compose 容器

打开 Certbot 改过的文件 `sudo nano /etc/nginx/sites-available/newme`，找到 `server { listen 443 ssl; ... }` 那一段，把里面的 `location /` 改成：

```nginx
    location / {
        proxy_pass http://127.0.0.1:37200;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }
```

保存，`sudo nginx -t && sudo systemctl reload nginx`。

验证：

```bash
curl -fsS https://api.newme-app.com/api/v1/health
```

返回 200 JSON，HTTPS 就跑通了。

## 9. 前端（Expo Web 和 iOS/Android）怎么发出去

项目的前端是 Expo，分三种分发渠道，按优先级：

### 9.1 Expo Web（浏览器版）部署到 OSS + CDN

1. 本地 Mac 上打静态包：

   ```bash
   cd /Users/shenxi/Desktop/WORK-SPACE/newMe

     pnpm --filter @newme/mobile exec expo export --platform web
   ```

   产物在 `apps/mobile/dist/`。

2. 阿里云开一个 **OSS Bucket**：

## 10. 数据库备份、日志与监控

### 10.1 每天自动备份 Postgres

```bash
sudo mkdir -p /var/backups/newme
sudo chown newme:newme /var/backups/newme

cat > ~/newme/scripts/backup-db.sh <<'EOF'
#!/usr/bin/env bash
set -euo pipefail
STAMP=$(date +%Y%m%d-%H%M%S)
OUT=/var/backups/newme/newme-$STAMP.sql.gz
docker compose -f /home/newme/newme/docker-compose.yml exec -T postgres \
  pg_dump -U newme newme_prod | gzip > "$OUT"
# 保留最近 14 天
find /var/backups/newme -name 'newme-*.sql.gz' -mtime +14 -delete
EOF
chmod +x ~/newme/scripts/backup-db.sh
```

加到 cron：

```bash
crontab -e
# 加一行（每天凌晨 3 点跑）：
0 3 * * * /home/newme/newme/scripts/backup-db.sh >> /var/log/newme-backup.log 2>&1
```

**强烈建议**再开一个阿里云 OSS Bucket 专门存备份，每天把 `.sql.gz` 传上去，这样服务器挂了数据也还在。

### 10.2 看日志

```bash
cd ~/newme
docker compose logs -f --tail=200 api      # API 日志
docker compose logs -f --tail=200 nginx    # 内层 nginx
sudo tail -f /var/log/nginx/access.log     # 宿主机 nginx
```

容器日志大小已经在 `/etc/docker/daemon.json` 里限制过了（10MB × 3）。

### 10.3 基础监控

阿里云自带**云监控**（免费），控制台搜「云监控」→「主机监控」，能看到 CPU、内存、带宽、磁盘。在「报警规则」里加几条：

- CPU > 85% 持续 5 分钟
- 内存 > 90% 持续 5 分钟
- 磁盘使用率 > 80%
- 公网出带宽 > 2.5 Mbps（防跑流量）

报警方式选「短信 + 邮件」，绑定你的手机号。

## 11. 以后每次改完代码怎么更新上线

把流程固化成脚本，减少出错。

### 11.1 方案：SSH + Git pull + rebuild

前提是代码已经托管到 GitHub/Gitee，并且服务器上 `~/newme` 就是 git clone 来的。

```bash
# 服务器上
cat > ~/newme/scripts/deploy.sh <<'EOF'
#!/usr/bin/env bash
set -euo pipefail
cd /home/newme/newme
git fetch origin
git reset --hard origin/main
docker compose up -d --build
docker compose ps
curl -fsS http://127.0.0.1:37200/api/v1/health && echo "  health ok"
EOF
chmod +x ~/newme/scripts/deploy.sh
```

以后每次发版：

```bash
# 本地把改动推到 main 之后
ssh -i ~/.ssh/newme-prod.pem newme@<公网IP> 'bash /home/newme/newme/scripts/deploy.sh'
```

### 11.2 进阶：GitHub Actions 自动部署

在仓库加 `.github/workflows/deploy.yml`，push 到 main 自动 SSH 上去跑 `deploy.sh`。把 SSH 私钥作为 `secrets.SERVER_SSH_KEY` 配进去。需要再告诉我，我给你写这份 workflow。

### 11.3 回滚

```bash
cd ~/newme
git log --oneline -n 10                 # 找到上一个好版本的 commit
git reset --hard <commit-hash>
docker compose up -d --build
```

Postgres 数据如果要回滚，解压最近的备份：

```bash
gunzip -c /var/backups/newme/newme-xxxx.sql.gz \
  | docker compose exec -T postgres psql -U newme newme_prod
```

## 13. 常见问题排查（FAQ）

**Q1：`docker compose up` 卡在 `pnpm install`，几分钟没反应。**
A：走镜像加速器。确认 `/etc/docker/daemon.json` 里填了阿里云的 mirror 地址并重启过 docker。

**Q2：浏览器访问 `http://<公网IP>` 超时。**
A：依次检查：①阿里云控制台「安全组」是否放开了 80/443 入方向；②服务器 `sudo ufw status` 是否也放开；③`docker compose ps` 看 nginx 容器是不是 Up；④`curl http://127.0.0.1:37200/api/v1/health` 本机先通。

**Q3：`docker compose up` 报 `Cannot connect to the Docker daemon`。**
A：`sudo systemctl start docker`；如果你没加进 docker 组，临时在命令前加 `sudo`。

**Q4：Postgres 容器起不来，日志里写 `FATAL: password authentication failed`。**
A：最常见原因是之前用旧密码建过 volume。容器数据持久在 `postgres_data` volume 里，改密码不会改旧用户的密码。两种处理：①进容器用 `ALTER USER newme PASSWORD '...'` 改；②开发阶段可以 `docker compose down -v` 把 volume 清掉重建——**上线后千万不要这么做**，会清掉所有数据。

**Q5：备案期间能先用 HTTP 测试吗？**
A：国内服务器 80/443 在备案前是被运营商拦的。可以换**新加坡/香港**节点的 ECS 先测，或者 SSH 开隧道本机测。

**Q6：磁盘快满了。**
A：`df -h` 看分区，常见大户是 Docker 的未使用镜像和日志。清理：

```bash
docker system df                     # 看占用
docker system prune -a --volumes     # 慎用！会删未被使用的镜像和卷
docker compose logs --tail=0 -f      # 只看新日志
```

**Q7：`certbot` 申请证书失败 `DNS problem`。**
A：说明域名还没真正解析到这台 IP。`dig api.newme-app.com` 看看返回，等 DNS 生效（可能要 10 分钟）再重试。

**Q8：我想换一台更大的服务器。**
A：ECS 支持「变配」——在控制台选实例 → 更多 → 升降配。升配不用重装系统，几分钟就能完成，期间会重启一次。数据和配置都保留。
